# visualization

## 目的

这份规范定义 `ViroScope OS` 中系统发育树的默认可视化标准。

目标不是“把树画出来”，而是输出：

- 清晰
- 可解释
- 可投稿
- 可重复

的图。

## 当前适用范围

第一阶段只覆盖：

- Influenza A
- H3N2
- HA 基因
- IQ-TREE 输出的 ML 树

## 总体风格

默认采用：

- 简洁
- 专业
- 留白平衡
- 接近 `Nature` / `Science` / `Cell` / `Nextstrain` 风格

避免：

- 过度装饰
- 霓虹色
- 过粗线条
- 过密标签
- 花哨背景

## 默认布局

默认使用：

- 横向矩形系统发育树

不默认使用：

- radial
- circular

要求：

- 保持分支长度比例
- 对齐 tip labels
- 自动调整间距
- 避免标签交叉

## 分支样式

默认规则：

- 主分支：深灰色
- 终末分支：略细
- 根分支：可略粗

建议参数：

- 主分支宽度：`0.6 pt`
- 终末分支宽度：`0.5 pt`

不建议把整棵树画得过黑或过粗。

## 节点与 Bootstrap

默认规则：

- 只显示 `>=70` 的 bootstrap
- `<70` 不在主图强调

文字建议：

- 字体：Arial 或 Helvetica
- 大小：`6-7 pt`
- 颜色：黑色

原则：

- 节点信息可读，但不能淹没主图结构

## 标签规则

推荐标签格式：

- `A/Location/Number/Year`

例如：

- `A/Tokyo/21/2025`
- `A/Guangdong/152/2024`

默认不显示：

- 长 accession 号

除非研究问题明确需要。

字体建议：

- Arial 或 Helvetica
- `7-8 pt`

要求：

- 标签尽量对齐
- 避免重叠
- 不因标签过长破坏版面

## 颜色规则

颜色必须：

- 色盲友好
- 对比清晰
- 专业克制

默认语义分配：

- `J.2.1`：蓝色
- `J.2.2`：红色
- `K`：绿色
- `L`：橙色
- WHO 疫苗株：紫色
- 参考株：黑色
- 用户样本：金色
- 背景样本：灰色
- 未知或未定义谱系：浅灰色

如果将来谱系增多，优先保持：

- 主谱系高区分度
- 次要谱系低饱和度

## 高亮规则

优先高亮以下对象：

- WHO 疫苗株
- 当前疫苗株
- 关键参考株
- 最早分离株
- 最新分离株
- 中国样本
- 用户自有样本

允许的高亮方式：

- 粗体标签
- 彩色圆点
- 彩色分支
- 略粗分支

原则：

- 只高亮少数关键对象
- 不允许全树都在强调

## 元数据注释

如果元数据完整，推荐增加 annotation strips。

优先顺序：

1. Country
2. Collection year
3. Clade
4. Vaccine strain
5. User sequence
6. Host
7. Sampling region

样式要求：

- 简单色条
- 不喧宾夺主
- 与主树留出清楚间距

## 时间信息规则

如果采样日期可靠且研究问题涉及时间变化，可以建议时间树。

否则：

- 默认保持 ML 树

不要为了“高级”而默认改成时间树图。

## Scale Bar

要求：

- 显示分支长度比例尺
- 放在左下或不干扰主体的位置
- 不做额外强调

## 定根说明

图中或图例中必须说明：

- 使用外群定根

或

- 使用 midpoint rooting

不允许根的处理方式不写明。

## 图例要求

每张主图至少说明：

- 树构建方法
- 软件
- bootstrap 显示阈值
- 颜色含义
- 疫苗株与参考株标记方式
- scale bar 含义
- 如有色条，说明各元数据色条含义

## 输出格式

默认导出：

- SVG
- PDF
- 600 dpi PNG

要求：

- 字体可嵌入
- 矢量图可编辑

## 工具建议

### iTOL

适合：

- 快速对齐标签
- 分支着色
- 元数据色条
- 导出 SVG

### FigTree

适合：

- 基础树形调整
- branch width
- scale bar
- tip alignment

### Adobe Illustrator

只用于：

- 轻微标签微调
- 图例对齐
- 颜色统一

不允许：

- 修改拓扑
- 修改分支长度
- 修改 bootstrap

## 质量检查

出图前至少检查这些问题：

- 标签是否重叠
- 颜色是否过多
- 高亮是否过量
- 疫苗株是否明确
- 用户样本是否明确
- scale bar 是否存在
- 定根方式是否说明
- 图在缩小后是否仍可读

## 第一版最低标准

第一版可视化模块只要做到这些就够：

- 能读取树文件和元数据
- 能输出横向矩形树
- 能高亮疫苗株与用户样本
- 能显示 `>=70` bootstrap
- 能导出 SVG 与 PNG

## 当前不做

这一版先不规定：

- 自动矢量排版优化
- 复杂交互式网页树
- 3D 可视化
- 动态时间动画

这些都不是第一阶段必需品。
