# PRD

## Why

现在很多病毒系统发育分析工作都散落在脚本、网页工具、笔记本和手工作图里。

结果是：

- 难以复用
- 难以解释
- 难以标准化
- 难以长期积累

`ViroScope OS` 的目标，是把这些零散步骤整理成一套可重复、可扩展、可持续维护的科研系统。

## Problem

这个项目要解决的核心问题是：

- 如何规范整理序列和元数据
- 如何稳定地完成系统发育分析
- 如何自动标注谱系、突变和疫苗株
- 如何输出可投稿的系统发育图
- 如何让AI参与解释，但不输出超出证据范围的结论

## MVP

`v0.1` 不是软件产品，而是项目模板。

它至少要包含：

- 项目定义
- 开发路线图
- 模块架构
- 科研规范目录
- Prompt目录与知识库目录

`v0.2` 到 `v0.6` 的目标，是跑通第一条真实可用的流感流程：

1. 元数据整理
2. QC
3. 序列比对
4. ML建树
5. 注释
6. 论文级可视化

## Out Of Scope

第一版明确不做这些内容：

- 多病毒通用平台
- 云端平台
- Web界面
- 自动写整篇论文
- 默认使用BEAST时间树
- 复杂多Agent系统

## Target User

第一用户画像：

- 病毒学研究者
- 生物信息初学者
- 有明确科研问题，但不想先陷入软件工程细节的人

## Success Criteria

项目达到以下标准，就算第一阶段成功：

- 仓库结构稳定
- Codex能按文档持续扩展模块
- 能完整跑通一次 `H3N2 HA` 分析
- 能输出一张清晰的系统发育图
- 能输出一段简洁、不过度解释的结果摘要

## First Milestone

`Milestone 1`：完成项目模板，并把模板作为后续开发合同。
