> **重要性：所有 AI Agent 必须遵守此规范。这是整个系统科学正确性的保障。**

## 1. 采样规范 (Sampling)

### 全局系统发育树

| 参数 | 规范 | 原因 |
|------|------|------|
| 样本量 | **300–500** | 超过 500 IQ-TREE 速度骤降；低于 300 全球代表性不足 |
| 时间跨度 | ≥5 年 | 保证进化信号 |
| 地理覆盖 | ≥3 大洲 | 避免地理偏倚 |
| 每个国家/年份 | ≤5 条 | 防止单一地点过度代表 |
| 参考株 | 必须包含 WHO 疫苗株 | 作为抗原锚点 |

### 亚谱系树 (Subclade)

| 参数 | 规范 | 原因 |
|------|------|------|
| 样本量 | **50–120** | 谱系内多样性有限 |
| 时间跨度 | ≥3 年 | — |
| 包含 Outgroup | 谱系外最近的 3–5 条 | 确定根位置 |

## 2. Outgroup 选择规范

### 什么时候**必须**使用 Outgroup

- 新建亚谱系树（需要确定根）
- 跨亚型比较（如 H3 vs H1）
- 跨宿主比较（如禽 vs 人）

### 什么时候**不需要** Outgroup

- 全球树（Midpoint root 已足够）
- 纯疫苗株比较（谱系明确）
- 抗原位点突变热力图（不需要根）

### Outgroup 选择标准

1. 与目标谱系最近的姐妹谱系
2. 3–5 条序列（不是 1 条）
3. 覆盖相似时间范围
4. 标注为 outgroup（不在最终图中显示）

## 3. 建树规范

### ML Tree（IQ-TREE）

| 场景 | 模型 | Bootstrap |
|------|------|-----------|
| 常规进化分析 | ModelFinder + 最佳模型 | ≥1000 |
| 快速预览 | GTR+F+I+G4 | 500 |
| 大规模筛选 | -fast | 关闭 |

### Time Tree（BEAST / TreeTime）

| 场景 | 方法 | 注意 |
|------|------|------|
| 进化速率估算 | BEAST | 需要采样日期 |
| 谱系起源时间 | TreeTime | 比 BEAST 快但精度低 |
| 疫苗株更新周期 | 两者交叉验证 | 严格分子钟 |

### 什么时候用 ML Tree，什么时候用 Time Tree？

| 问题 | 推荐 |
|------|------|
| "哪些序列聚在一起？" | ML Tree |
| "这个谱系什么时候出现？" | Time Tree |
| "抗原位点突变在树上如何分布？" | ML Tree + Annotation |
| "进化速率是否加快？" | Time Tree (BEAST) |

## 6. 结论规范 (Interpretation Guidelines)

### 系统发育树**能**支持

- 谱系划分（Clade assignment）
- 序列间相对亲缘关系
- 疫苗株与流行株的遗传距离

### 系统发育树**不能**直接支持

- 因果关系（如"突变 X 导致传播增加"）
- 抗原性变化（需要结合 HI 实验数据）
- 毒力变化（需要动物实验）

### 需要结合其他数据验证

- 抗原漂移 → 需 HI 实验或 FRA/glycan 分析
- 跨宿主传播 → 需流行病学数据
- 疫苗有效性 → 需人群血清学数据
